# Quickstart

1. Read the paper in `papers/`.
2. Review the architecture diagrams in `diagrams/`.
3. Read the mathematical notes in `mathematics/`.
4. Review experiment templates in `experiments/`.
5. Explore the engineering notes in `engineering/`.
6. Follow the roadmap in `future-research/`.

## Viewing Mermaid Diagrams

GitHub renders Mermaid diagrams in Markdown automatically when they are inside fenced code blocks marked `mermaid`.

The `.mmd` files in `diagrams/` contain reusable Mermaid source. You can view them by:

- opening them directly on GitHub;
- copying the content into a Mermaid-compatible editor;
- embedding the diagrams in Markdown using fenced `mermaid` blocks.
